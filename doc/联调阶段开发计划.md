# WinAudioBridge 联调阶段开发计划

## 1. 当前目标

当前进入 Windows 端与 Android 端的首次联调阶段，目标是验证最小可用链路：

- Windows 端可成功选择目标设备
- Windows 端可建立 ADB 端口转发
- Android 端前台服务可在后台持续监听 `5000` 端口
- Windows 端可发送 `SessionInit`
- Android 端可正确解析 `SessionInit`
- Windows 端可发送 `AudioFrame`
- Android 端可收到并写入 `AudioTrack`
- Android 端音量调节可即时生效

## 2. 联调顺序

### 第 1 步：环境确认

联调前先确认：

- Android 设备已开启 USB 调试
- `adb devices` 可看到目标设备
- Android 端安装的是当前 `AudioBridge` Debug 包
- Android 端应用包名为 `dev.ran.audiobridge`
- Windows 端设置中的包名与之保持一致

### 第 2 步：Android 端服务验证

先单独验证 Android 端：

- 手动启动后台服务
- 确认通知栏出现前台服务通知
- 应用退到后台后服务仍在运行
- 音量滑块拖动可更新界面状态

验收点：

- 服务不会立即退出
- 通知栏持续存在
- UI 状态可见

### 第 3 步：Windows 端链路准备验证

在 Windows 端依次验证：

- 设置页设备列表正常刷新
- 能识别目标 Android 应用运行状态
- 点击“准备链路”后状态变为已就绪
- 已建立 `tcp:5000 -> tcp:5000` 端口转发

验收点：

- 主界面显示目标设备
- 主界面状态进入“已就绪”
- 无异常报错

### 第 4 步：`SessionInit` 联调

在 Windows 端点击“开始推流”前，重点看 Android 端是否收到初始化消息。

需要验证：

- Android 端连接状态变为已连接
- Android 端状态变为“收到 SessionInit，播放参数已初始化”
- UI 中显示当前编码、采样率、声道、位深、Buffer
- `AudioTrack` 初始化成功

验收点：

- `SessionInit` 解析成功
- Android 端未崩溃
- 前台服务仍保持运行

### 第 5 步：`AudioFrame` 联调

在 `SessionInit` 成功后继续验证 PCM 数据播放。

需要验证：

- Windows 端开始发送音频帧
- Android 端 `lastSequence` 持续递增
- Android 端状态进入“后台播放中”
- 实际可听到声音

验收点：

- `AudioFrame` 能持续接收
- `AudioTrack.write()` 无明显错误
- 播放无立即中断

### 第 6 步：音量与后台验证

在音频可播放后验证：

- 调整 Android 端音量滑块后声音立即变化
- 应用退到后台后播放不中断
- 通知栏仍显示前台服务
- Windows 停止推流后 Android 返回等待连接状态

验收点：

- 音量控制生效
- 后台播放稳定
- 停止推流后状态恢复正确

## 3. 联调时优先排查的问题

如果联调失败，优先按以下顺序排查：

1. **设备不可见**
   - 检查 `adb devices`
   - 检查 USB 调试授权

2. **Windows 端显示未运行目标应用**
   - 检查 Android 应用是否已启动一次
   - 检查包名 `dev.ran.audiobridge` 是否一致

3. **准备链路失败**
   - 检查端口 `5000` 是否被占用
   - 检查 Android 前台服务是否已启动

4. **Android 未收到 `SessionInit`**
   - 检查 Windows 是否真的进入“开始推流”
   - 检查 Android `ServerSocket` 是否仍在监听

5. **收到 `SessionInit` 但无声音**
   - 检查 `AudioTrack` 初始化参数
   - 检查编码是否为 `PCM16`
   - 检查音量是否过低

6. **后台退出或播放中断**
   - 检查前台服务是否被停止
   - 检查通知栏是否正常存在

## 4. 联调完成后的下一轮开发

当首轮联调完成后，下一步开发按以下优先级推进：

### 优先级 A

- 修正协议字段不一致问题
- 修正 `AudioTrack` 参数映射问题
- 修正断线恢复问题

### 优先级 B

- 增加 Windows 端日志面板
- 增加 Android 端最近错误信息展示
- 增加连接超时与自动重试

### 优先级 C

- 增加通知栏停止播放动作
- 增加状态回传预留
- 优化 UI 与状态提示

## 5. 本轮预期产出

本轮联调后，希望得到以下结果：

- 是否成功建立 Windows -> Android 音频链路
- `SessionInit` 是否成功
- `AudioFrame` 是否成功
- 后台播放是否稳定
- 音量调节是否正常
- 当前最关键的失败点在哪一层

以上结论将作为下一轮修复与优化的输入。
