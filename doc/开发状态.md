## 当前风险/阻塞

- 当前已完成骨架实现，但尚未与 Android 接收端完成真实链路联调
- Android 应用包名当前已默认对齐为 `dev.ran.audiobridge`，仍需在联调时确认未被用户本地配置覆盖
- 新增远程音量控制需求后，当前通信协议需要从单向音频流扩展为双向控制/状态同步链路
- Windows 应用图标提取与会话聚合存在兼容性差异，需要在常见应用上专项验证
- 某些 Windows 应用会话可能没有可提取 icon 或返回系统默认 icon，仍需在常见应用上验证效果
